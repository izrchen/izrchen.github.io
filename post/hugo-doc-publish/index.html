Hugo博文发布

博文发布流程
June 9, 2022 · 1 min · 114 words · Nathan

总体流程#

org to markdown(本地) to markdown(仓库)

操作1：org to 本地markdown#

从org文件夹中，将 博文.org 导出到hugo目录 hugo/content/post/ 下 博文.md

文件头（生成）#

  • 相关配置

    #+HUGO_BASE_DIR: <导出文件的保存路径>
#+HUGO_SECTION: post
#+TITLE: <文章标题>
#+DESCRIPTION: <文章描述>
#+HUGO_CATEGORIES: <文章分类>
#+HUGO_TAGS: <文章标签>
#+HUGO_SLUG: <网页访问路径>
#+HUGO_DRAFT: true
#+AUTHOR: <作者>
#+DATE: "2022-01-01"

  • 快捷键

    • 输入 hmatter ，再按 TAB

内容写作#

导出（命令）#

  1. C-c C-e
  2. H h

操作2：markdown 本地预览#

在 hugo 根目录的命令行输入： hugo server -D

操作3：markdown to html#

在 hugo 根目录的命令行输入： hugo

操作4：html 发布到仓库#

在 hugo/public 下的git终端输入：

git add .
git commit -m "备注"
git push origin main

其他操作#

博文图片嵌入#

方案一：截图图传#

  • 图传网站：https://sm.ms/
  • 提取图片链接，可使用JS代码提取
  • 插入HTML标识，key： htmlpic
#+CAPTION:
#+ATTR_HTML: :width $2% :align center
picture url

方案二：SVG#

  • svg在线编辑：https://svg.wxeditor.com/
  • 将svg图片保存到本地
  • 复制svg代码到org或者md文件中，根据需要进行修改
  • 为保证md文件中的svg原样输出到html，需在目标代码的前后插入 {{rawhtml}} 和 {{/rawhtml}}
"两个花括号" <rawhtml> "两个花括号"
目标代码
"两个花括号" </rawhtml> "两个花括号"

问题及解决#

博客list中博文重复出现#

  • 因为标签变更，旧标签还在
  • 文件内容相同，更改了文件名，由于slug一样，则出现两条重复路径的博文
